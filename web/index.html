What is Onionoo?

Onionoo is a web-based protocol to learn about currently running Tor relays and bridges. Onionoo itself was not designed as a service for human beings—at least not directly. Onionoo provides the data for other applications and websites which in turn present Tor network status information to humans.


What Onionoo clients are available?

There are currently two Onionoo clients in the making:

  • Android app: An Android app will let you add your relays or bridges to your favorites list and keeps you always informed whether they are running or not.
  • Tor network status website: A JavaScript-only Tor network status website will enable you to search for relays by nickname, fingerprint, or IP address and sort and filter results by more criteria.

Stay tuned until these clients are available. Until then, your only chance to learn what Onionoo does is to start developing your own client.


How can I develop a new Onionoo client?

Don't want to wait until the clients above are available? Want to implement your own and tell us to add it to the list above? Here are a few ideas:

  • Vidalia or arm extension: Extend Vidalia and/or arm to look up details for the bridge that the user is running and display what pool the bridge is contained in.
  • Social network site plugin: Add a plugin to the social network site of your choice to show your friends what Tor relays and bridges you're running and how that helps users around the world.
  • Desktop tray icon: Write a tray icon for your favorite desktop environment that tells you when your relay or bridge is down and that displays some basic usage statistics.
  • E-mail notification service: Improve our e-mail notification service Weather by implementing its own relay search or extending it to report when a bridge drops off the network.
  • Command-line tool: Implement a command-line tool that quickly searches a relay or bridge and prints out some status information to help debug problems.

Learn more about the Onionoo protocol to implement your own Onionoo client.


How can I help develop the Onionoo server or run my own?

The Onionoo server is written in Java with a tiny portion of Java Servlets. The source code and issue tracker are available here. Any help with the Onionoo server code is much appreciated!

While it's not required for running or developing an Onionoo client, anyone can run their own Onionoo server. No special Tor developer privileges are required for that. Instructions for setting up the Onionoo server to fetch the required data from the Tor servers is described in the INSTALL file in the sources.
